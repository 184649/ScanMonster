#### 同種族の別個体

将来の別個体は、ベース種族の下にぶら下がる派生として扱います。

例:

- `Dog` の別個体: Chihuahua / Shiba Inu / Bulldog
- `Cat` の別個体: Calico Cat / Siamese Cat / Persian Cat
- `Carp` や `Goldfish` の品種差、植物の栽培品種、地域型、色違い

これらは新しいベース種族としてワールド100体に数えません。実装時は `parentSpeciesId` または `parentSpeciesEn` を持つ別個体マスターを作り、発見済みベース種族の中で姿・品種・地域型として解放します。

## カテゴリ拡張方針

初回リリースの `HabitatGroup` は以下です。

- `land`
- `water`
- `sky`
- `bug`
- `reptile`
- `rare_world`

将来カテゴリを増やす場合は、以下を満たす必要があります。

- 既存ユーザーの解放状態を壊さない
- 解放コスト表を再設計する
- 気配ブースト率が自然に分配される
- レア確率1〜3%の方針を維持する
- DPでレア確率を上げない

## 世界拡張の候補

| 世界 | 初回 | 将来案 |
| --- | --- | --- |
| animal | 対象 | 現在の主軸 |
| food | 対象外 | OCRや商品名取得なしで扱える設計が必要 |
| plant | 対象外 | 写真認識は初回では使わない |
| dinosaur | 対象外 | イベント向け |
| fantasy | 一部レアのみ | レアイベント向け |
| space | 対象外 | 時刻/方角連動は将来検討 |

## 拡張時の原則

- MVPのスキャン体験を壊さない
- 生コード値を保存しない
- 正確な位置情報を保存しない
- ユーザーが「課金しないと進まない」と感じる設計にしない
- 既存の図鑑/DP/称号と矛盾させない
- まずfeature flagで段階公開する
