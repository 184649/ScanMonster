## Recommendation: 24

Start the **new IP release candidate portfolio at 24**, subject to Gate 6 success. The strongest current risk is not roster breadth but whether a distinctive, reproducible IP language exists. Twenty-four gives each character more silhouette, model-sheet, small-size, and merchandise validation time while leaving twelve obvious expansion slots to 30 and 36.

The 24 target should include all four core worlds and one legendary concept per world so that the normal-completion mechanic has a coherent destination. The proposed 16/4/4 split is a design target; actual unlock totals must be reviewed against repetition data and production capacity before any future release-status change. Secret remains unadvertised and does not fill a visible quota.

Mitigations for a smaller pool:

- preserve unseen weighting and meaningful rediscovery/number badges;
- stagger world access and additions without changing old discovery records;
- add in reviewed waves of three to six, not a bulk dump;
- use analytics on repeat interval, first-session completion, and name recognition before moving from 24 to 30;
- never renumber or recycle a legacy ID when adding a new identity.

## Relationship to current 89

No current entry is deleted, reclassified, renamed, or demoted in Phase 2A. The 24 proposal is a future **IP production and presentation portfolio**, not an edit to the existing canonical initial list. The 89 can coexist as:

- `SpeciesMotif` sources and zoological profiles;
- legacy catalog entries with permanent history/number bindings;
- future IP-development candidates;
- displayable zoological content under `zoological`/`hybrid` mode;
- explicit legacy-to-new mappings after separate approval.

Any later change to `releaseStatus`, catalog population, or seed must be a separately reviewed migration with discovery-number equivalence checks and rollback. This document performs none of those actions.
