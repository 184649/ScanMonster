# WORLDAWN 自然史図鑑イラスト — 共通アートディレクション

図鑑特化版の正式イラスト方針。**旧キャラクター版（マスコット調）とは別系統**であり、旧プロンプト・旧画像は `docs/legacy-character-prompts/` と `assets/characters/` に保存したまま流用しない。

## 位置づけ

| | 旧キャラクター版 | 図鑑特化版（この文書） |
|---|---|---|
| 方向 | マスコット・キャラクター性 | 科学的正確さを基盤とした自然史イラスト |
| プロンプト | `docs/legacy-character-prompts/` | `docs/encyclopedia-prompts/` |
| 画像 | `assets/characters/` | `assets/encyclopedia/` |
| 検査 | `npm run validate:release-assets`（従来基準） | `npm run validate:encyclopedia`（1024・真の透明画素） |

**旧資産は削除・上書きしない。** 新旧を混在させず、ディレクトリとプロンプトで分離する。

## 図鑑分類ごとのテンプレート

| 分類 | 対象 | テンプレート |
|---|---|---|
| NORMAL | 現生生物の通常個体・通常形態 | [01_NORMAL.md](01_NORMAL.md) |
| RARE | アルビノ・白変種・黒化個体など**実在が確認された**希少形態 | [02_RARE.md](02_RARE.md) |
| LEGEND | 絶滅した実在生物 | [03_LEGEND.md](03_LEGEND.md) |
| SECRET | 神話・伝承・空想・SF上の生物 | [04_SECRET.md](04_SECRET.md) |

**虫・節足動物は [05_ARTHROPODS.md](05_ARTHROPODS.md) を必ず併用する**（分類テンプレートより優先）。

一覧・発見演出・共有導線の設計は [COLLECTION_EXPERIENCE.md](../COLLECTION_EXPERIENCE.md)。

図鑑分類は `assets/characters/character-classification.json` の `dexClass` が正本。**この分類は表示専用であり、出現確率・抽選・解放条件へは一切影響しない。**

## 全分類で共通の方針

正は「**種の特徴が正しく分かる、見やすく整理された高品質な生物イラスト**」。

**リアルイラストだが、生々しくしない。** 細密さで勝負しない。

- 対象種を一目で識別できる骨格・体型・顔・脚数・指・翼・鰭・触角・体節。
- 毛・羽・鱗・皮膚・甲殻の質感は**描き分ける**が、質感の作り込み自体を主役にしない。
- 陰影は多段階だが整理された面で。毛の一本描写のような高周波テクスチャに頼らない。
- **一覧に並べたときの統一感を最優先**：サイズ感、向き、光源、描画密度を全種で揃える。

### 禁止（全分類共通）

**生々しさの禁止**

- 写真のような超写実
- マクロ写真風（極端な寄り・被写界深度・質感の作り込み）
- 昆虫標本写真のような生々しさ
- 濡れた質感、粘液、体液、傷、捕食中の描写

**キャラクター化の禁止**

- キャラクター化、アニメ化、擬人化
- 過度なデフォルメ、巨大な丸目、人間的な笑顔・眉・まつ毛
- 架空の角、ドリル、宝石、鎧、装備、発光、植物装飾

**演出の焼き込み禁止**

- 王冠・オーラ・エフェクトなど**レアリティを画像へ焼き込む表現**
  特別感は**UIフレーム・発見演出・共有導線**で作る（[dexPresentation.core.ts](../../src/services/dexPresentation.core.ts)）
- 文字、ロゴ、UI、フレーム、キャプション、透かし、署名
- 地面、台座、影、風景、比較線、説明パネル
- 既存の有名IPキャラクターの模倣

### 一覧での統一（重要）

図鑑一覧は3列グリッドで、サムネイルは正方形に並ぶ。次を全種で揃える。

| 項目 | 指定 |
|---|---|
| 向き | 自然で識別しやすい3/4方向。左右はどちらでもよいが種内で統一 |
| 画面占有率 | 被写体が画面の 60〜80% を占める。極端に小さく・大きくしない |
| 光源 | 左上前方から。種ごとに変えない |
| 描画密度 | 顔と識別特徴を最も丁寧に、他は情報量を落とす |
| 姿勢 | 立ち姿・自然な静止。極端な動作や俯瞰・アオリにしない |

一覧で並べたときに**同じシリーズに見えること**が、細密さより優先される。

### 出力仕様（全分類共通・厳守）

- **1024×1024ピクセル、正方形、PNG**
- **本物の透明背景**（アルファチャンネルがあるだけでは不可。実際に透明な画素が存在すること）
- 単色背景を残さない。透明を表す市松模様を描かない。足元に不透明な楕円影を置かない
- 生きものは**1体のみ・全身・中央配置**
- 主要部位（頭・四肢・尾・角・翼など）を画面外で切らない
- 種名・学名・特徴文は**後工程でUIが配置する**。画像へ焼き込まない

保存先は `assets/encyclopedia/<dexclass>/<id>.png`（例：`assets/encyclopedia/normal/ground_elephant.png`）。

## 生成後に必ず実行する検査

```
npm run validate:encyclopedia
```

自動で判定するのは、寸法・PNG健全性・RGBA・**alpha最小値<255**・**透明画素数>0**・四隅の焼き込み・外周の見切れ・被写体面積。

終了コードは `0`=全件合格 / `1`=不合格あり / `2`=未生成。**`2` を合格として報告しない。**

### 自動判定できない項目（目視で確認する）

1. 生きものが1体だけか（全身）
2. 対象種として識別できるか
3. 文字・ロゴ・台座・風景が写り込んでいないか
4. 実在生物にファンタジー装飾が付いていないか
5. RAREが通常種と同じ骨格か／LEGENDが古い復元になっていないか

**目視していない画像を「品質合格」と報告しない。**

## 科学情報との関係

図鑑本文（分類・分布・保全状況など）は**画像生成AIに書かせない**。`assets/characters/species-profiles.json` に構造化データとして持ち、人間の確認（`reviewStatus: confirmed`）を経たものだけ公開する。

プロンプトへ差し込む `{diagnosticFeatures}` などの値も、このファイルの確認済みデータから取る。**未調査の項目を推測で埋めてプロンプトへ入れない。**
