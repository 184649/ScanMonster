# WORLDAWN 初回リリース設計

最終更新: 2026-07-05

> **構造**: 分類は「領域（RealmGroup） > ワールド（WorldGroup） > キャラクター」。**DPで解放する単位はワールド**。初回は生物領域の4ワールド（地上／水辺／空／虫）。正典は [WORLD_SYSTEM.md](WORLD_SYSTEM.md) §7。

## 方針

初回リリースでは、スキャン、発見、図鑑、DP、ワールド解放、ワールドブースト、称号、マイページ、設定に集中する。

以下は初回リリースに入れない。

- バトル
- 探索
- 研究
- アニメーション
- 個体値
- 個体差コレクション
- カテゴリ図鑑
- 商品名取得
- OCR
- 位置情報を使った出現制御
- 交換
- ランキング
- 課金
- 広告

## 基本ループ

```text
スキャン
↓
キャラクター発見 / 再発見
↓
DP獲得
↓
ワールド解放
↓
出現対象が増える
↓
ワールドブーストで狙うワールドを出やすくする
↓
図鑑と称号が進む
```

## 初回ワールド選択

初回開始時に、次の4ワールドから1つを選ぶ（すべて生物領域）。

- ground: 地上ワールド（地上の動物）
- waterside: 水辺ワールド（水・海・川・水辺の生き物）
- sky: 空ワールド（鳥・翼・飛行の生き物）
- bug: 虫ワールド（虫・小さな生き物）

選んだワールドは無料で解放済みになり、最初はそのワールドだけが出現対象になる。

## ワールド解放

ワールド解放コストは、ワールド種類ではなく「何個目の解放か」で決まる。

| 解放数 | コスト |
| --- | ---: |
| 1ワールド目 | 0 DP |
| 2ワールド目 | 1,000 DP |
| 3ワールド目 | 2,300 DP |
| 4ワールド目 | 4,200 DP |
| 5ワールド目 | 7,000 DP |
| 6ワールド目 | 11,000 DP |

## ワールド（WorldGroup）

```ts
type RealmGroup = "life" | "space" | "history" | "micro" | "food";
type WorldGroup =
  | "ground" | "waterside" | "sky" | "bug" | "scale" | "phantom"   // 生物領域（初回は ground/waterside/sky/bug）
  | "planet" | "constellation" | "bc" | "jomon" | "heisei"          // 宇宙・歴史領域（将来）
  | "atom" | "virus" | "staple_food" | "dessert";                   // ミクロ・食べ物領域（将来）
```

将来ワールドは定義のみ（初回は生物4ワールド）。レアキャラは各ワールドへ紐づける（ワールド別レア。sky は現状レア未整備＝将来追加）。

## 出現決定

1. `sourceHash + localDate` で同日同コード制限を確認
2. 有効スキャンなら抽選開始
3. 解放済みワールドを取得
4. ワールドブーストがあればワールド出現率を補正
5. ワールドをランダム抽選
6. レア確率ロール
7. 通常/レア候補を絞る
8. ワールド内からランダムに1体選ぶ
9. 未所持ならNEW発見
10. 所持済みなら再発見

未発見優先は入れない。所持済みキャラも通常候補に含める。

## レア

- 初心者: 約1%
- 進行後: 最大3%
- DPでレア確率を上げない
- ワールドブーストでもレア確率を上げない
- レア確率ロールはワールド抽選後に行う

## ワールドブースト

- コスト: 300 DP
- 対象: 解放済みワールドのみ
- 効果: 次の10回の有効スキャン
- 同時使用: 1つまで
- 同日同コードブロックでは残り回数を減らさない
- レア確率は変えない

4ワールド解放済みの水辺ブースト例:

- waterside 55%
- ground 15%
- sky 15%
- bug 15%

2ワールド解放済みの場合は対象ワールド70%、もう一方30%。

## 所持キャラ

同じキャラが出た場合は再発見として扱う。

```ts
type OwnedCharacter = {
  characterId: string;
  firstDiscoveredAt: string;
  lastDiscoveredAt: string;
  discoveryCount: number;
  favorite: boolean;
};
```

既存実装では `UserMonster` に互換フィールドを追加し、同じ `characterId` の発見では新規レコードを増やさず発見回数を更新する。

初回リリースの `characterId` はベース種族単位にする。チワワ、柴犬、ブルドッグのような同種族の別個体候補は、初回リリースでは独立キャラとして抽選・所持・図鑑登録しない。将来実装時は、親種族 `Dog` に紐づく別個体として扱い、ベース種族の発見数とは別の進捗にする。

## DP

- 新キャラ発見: 30 DP
- 通常キャラ再発見: 5 DP
- レアキャラ初発見: 100 DP
- レアキャラ再発見: 20 DP
- 今日の初スキャン: 20 DP
- 同日同コードブロック: DPなし

## 称号

称号はプレイ実績・自己表現のための機能。強さやレア確率には影響しない。

- スキャン回数
- 図鑑進捗
- ワールド解放
- ワールド別発見
- レア発見
- 再発見
- 連続スキャン

## 保存とプライバシー

- `rawBarcode` / `rawQrValue` / `normalizedValue` は永続保存しない
- 同日同コード制限は `sourceHash + localDate`
- 正確な位置情報は保存しない
- 位置情報を出現制御に使わない
